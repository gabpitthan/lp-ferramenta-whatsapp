# Research and Inspiration — Enterprise Benchmarking

## Objetivo
Pesquisar referências de nível enterprise para orientar decisões de design e copy. Zero referência genérica — cada fonte deve oferecer insight acionável específico para o contexto.

---

## Protocolo de Pesquisa

### Passo 1 — Moodboard Estratégico
Para cada projeto, coletar:
1. **3 referências de design** do Tier 1 com análise do elemento específico que converte
2. **1 referência cross-domain** inusitada com justificativa de aplicação
3. **1 anti-referência**: mostra o que NÃO fazer (concorrente genérico)

Formato de análise por referência:
```
Site: [url]
Elemento: [o que especificamente é notável]
Por que converte: [hipótese baseada em psicologia/UX]
Como adaptar: [implementação específica para este projeto]
```

### Passo 2 — Extração de DNA Visual
Ao analisar cada referência, extrair:
- **Paleta de cor**: valores hex principais + proporção de uso
- **Tipografia**: família + peso + tamanho relativo + letter-spacing
- **Espaçamento**: padrão de white space e densidade de informação
- **Animações**: tipo (fade, slide, scale) + duração + easing
- **Padrões de layout**: grid, breakpoints, hero structure

### Passo 3 — Pattern Library para Projeto
Compilar:
- 3 opções de hero layout (com sketch de estrutura)
- 2 opções de paleta (com razão emocional de cada)
- 2 combinações de fontes (com amostra visual de headline + body)
- Biblioteca de micro-interações aplicáveis

## Copy Research

### Análise de VOC (Voice of Customer)
Fontes para pesquisa de linguagem do cliente:
- Reviews do G2, Capterra, Trustpilot dos concorrentes
- Comentários do Reddit/LinkedIn sobre o problema que o produto resolve
- Fóruns e grupos do nicho de mercado
- Reviews da App Store/Play Store de produtos similares

**Extração de VOC:**
```
Frases reais do cliente sobre a DOR (usar textualmente no copy):
- "[citação]" — fonte
- "[citação]" — fonte

Frases reais sobre o DESEJO:
- "[citação]" — fonte

Objeções reais capturadas:
- "[objeção]" — fonte
```

### Competitor Copy Analysis
Para cada concorrente direto:
- Headline do hero
- Proposta de valor central
- CTAs usados
- Prova social apresentada
- Ponto cego (o que eles NÃO falam que você pode explorar)

## Avaliação de Referências (Score 1-10)

| Dimensão | Critério de 10/10 |
|---|---|
| **Diversidade** | Fontes de 3+ domínios diferentes (tech + art + cross-domain) |
| **Acionabilidade** | Cada referência tem implementação específica definida |
| **Relevância** | Contexto do projeto e público estão bem mapeados |
| **DNA Extraído** | Valores exatos (hex, px, ms) documentados, não genéricos |
| **VOC Capturado** | Frases reais do cliente documentadas como input para copy |

### Critério de Aprovação
- Mínimo 5 referências Tier 1 analisadas com DNA extraído
- Mínimo 1 referência cross-domain com justificativa clara
- VOC coletado de pelo menos 2 fontes externas com citações reais
- Pattern library com pelo menos 2 opções para cada decisão de design
