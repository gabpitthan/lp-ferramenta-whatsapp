# Conversion Copywriting — Copy de Alta Conversão

## Objetivo
Escrever copy que converte visitantes em leads/clientes usando metodologia VOC (Voice of Customer), frameworks de copywriting testados (AIDA, PAS, Before-After-Bridge) e princípios de psicologia comportamental. Zero hype genérico — copy baseado na dor real, linguagem real, prova real.

---

## Metodologia VOC (Voice of Customer)

O princípio mais poderoso do copywriting de conversão: **usar as palavras exatas do cliente**.

### Fontes de VOC
1. Reviews de concorrentes (G2, Capterra, Trustpilot, Google Reviews)
2. Comentários em posts de redes sociais sobre o problema
3. Entrevistas com clientes existentes (perguntas: "Como você descreveria o problema antes?")
4. Respostas de pesquisas NPS (principalmente detratores e promotores)
5. Transcrições de calls de vendas / chat de suporte
6. Grupos do Facebook, comunidades Slack/Discord do nicho
7. Fóruns Reddit relacionados ao problema

### Template de Coleta VOC
```
DORES (como o cliente descreve o problema, com suas palavras):
- "[citação textual]" — fonte: G2 review, @menção, entrevista
- "[citação textual]"

DESEJOS (o que o cliente quer alcançar):
- "[citação textual]"

RESULTADOS ESPERADOS (respostas tipo "o que mudaria com a solução?"):
- "[citação textual]"

OBJEÇÕES REAIS (o que impede de comprar agora):
- "[citação textual]"

LINGUAGEM PREFERIDA (termos que eles usam, não terminologia interna):
- Eles dizem "[termo do cliente]", não "[jargão corporativo]"
```

## Frameworks de Copy por Seção

### PAS — Problem, Agitate, Solution (para seção de problema)
```
P — Descrever o problema com precisão clínica (voz do cliente)
A — Agitar: consequências de não resolver (custo da inação)
S — Apresentar solução como alívio inevitável
```

**Exemplo:**
```
P: Você ainda perde vendas porque o follow-up chega tarde demais.
A: Cada hora sem resposta = cliente avaliando o concorrente. 
   Empresas que respondem em 5 minutos têm 21x mais chance de qualificar leads.
S: Com automação inteligente, você responde em segundos — mesmo fora do horário comercial.
```

### AIDA — para fluxo da página inteira
```
A — Attention: headline que quebra o padrão, gera curiosidade ou valida dor
I — Interest: benefícios que importam para o ICP (não features)
D — Desire: prova social + casos reais que tornam o desejo tangível
A — Action: CTA claro com urgência real + remoção de fricção
```

### Before-After-Bridge — para testimonials e case studies
```
BEFORE: Situação anterior (dor específica)
AFTER: Resultado após usar a solução (específico, mensurável)
BRIDGE: Como o produto chegou lá
```

## Regras de Copywriting Enterprise

### Regra 1 — Especificidade Bate Generalidade
- Evitar: "Aumente suas vendas"
- Usar: "Feche 34% mais contratos em 90 dias — garantido"
- Evitar: "Fácil de usar"
- Usar: "Configure em 48h sem precisar de TI"
- Evitar: "Muitas empresas usam"
- Usar: "+8.000 empresas brasileiras de 12 setores"

### Regra 2 — Benefício Antes de Feature (B2F — Benefits to Features)
Template: "[Resultado que o cliente quer] — [feature que entrega]"
- Errado: "API RESTful com 99.9% de uptime"
- Certo: "Integre com qualquer sistema sem depender de TI — nossa API RESTful garante 99.9% de disponibilidade"

### Regra 3 — Headline Formula (10 Opções, Escolher 1)
Gerar 10 headlines antes de escolher. Formatos comprovados:

```
1. [Número] + [período] + [resultado]: "Feche 3x mais contratos em 90 dias"
2. [Como + resultado sem dor comum]: "Como automatizar follow-up sem contratar SDR"
3. [Pergunta de dor]: "Cansado de perder vendas por falta de follow-up?"
4. [Declaração de transformação]: "De 12% para 78% de taxa de resposta em 3 meses"
5. [Credibilidade + resultado]: "O sistema que +8.000 empresas usam para dobrar vendas"
6. [Urgência + resultado]: "Pare de perder vendas agora — automatize em 48h"
7. [Oposição ao status quo]: "Chega de follow-up manual. Automatize e feche mais."
8. [Para quem é]: "Para gerentes comerciais que querem escalar sem contratar"
9. [Prova social + resultado]: "Como a TechVendas aumentou NPS de 32 para 71"
10. [Formato "Finalmente"]: "Finalmente: automação comercial que seu time realmente vai usar"
```

Critério de seleção: a headline que é mais específica E mais surpreendente vence.

### Regra 4 — CTAs que Convertem

**Fórmula:** [Verbo de ação forte] + [meu/minha] + [benefício específico]
- "Quero aumentar minhas vendas agora"
- "Gerar meu relatório gratuito"
- "Começar meu teste de 14 dias"
- "Reservar minha demonstração"
- "Ver como funciona para mim"

**Nunca usar:** Enviar, Clique aqui, Saiba mais, Cadastrar, Submit

**Micro-copy abaixo do CTA** (remove fricção):
- "Sem cartão de crédito. Cancele quando quiser."
- "Configuração em 48h. Suporte dedicado."
- "Já são +8.000 empresas. Junte-se agora."

### Regra 5 — Escrever para Escaneabilidade
80% dos usuários escaneiam, não leem. Estrutura de leitura:
- Primeira palavra de cada frase: a palavra mais importante
- Negrito: apenas no que é crítico (máximo 1 por parágrafo)
- Bullet points: máximo 5 por grupo, primeiro item mais importante
- Parágrafos: 2-3 linhas máximo
- Subheadings: a cada 150-200 palavras

## Checklist de Copy por Seção

### Hero
- [ ] Headline: clara em <5 segundos para um estranho?
- [ ] Subheadline: expande benefício com prova implícita?
- [ ] CTA: verbo específico + benefício + "meu/minha"?
- [ ] Micro-copy: remove a maior objeção imediata?
- [ ] Teste: alguém leigo entende o produto sem contexto?

### Features/Benefícios
- [ ] Cada item começa com BENEFÍCIO, não feature?
- [ ] Linguagem do cliente, não jargão técnico?
- [ ] Máximo 5 benefícios (lei de Hick)?
- [ ] Cada benefício tem 1 linha de prova ou dado?

### Prova Social
- [ ] Depoimentos com resultado específico e mensurável?
- [ ] Nome completo, cargo e empresa real?
- [ ] Foto real (não avatar)?
- [ ] Objeção respondida implicitamente pelo testimonial?

### Preço
- [ ] Ancoragem com plano mais caro primeiro?
- [ ] Comparação com alternativa cara para contextualizar valor?
- [ ] Money-back/garantia explícita?
- [ ] Micro-copy de segurança abaixo do CTA de compra?

### FAQ
- [ ] Objeções mais comuns dos calls de vendas respondidas?
- [ ] Tom confiante, não defensivo?
- [ ] CTA ao final do FAQ?

## Avaliação de Copy (Score 1-10)

| Dimensão | Critério de 10/10 |
|---|---|
| **Clareza** | Qualquer pessoa entende a proposta em <5 segundos |
| **VOC** | Linguagem do cliente, não jargão corporativo |
| **Especificidade** | Números, nomes, prazos reais — nada vago |
| **Força Emocional** | Ativa dor E desejo de forma ética |
| **CTAs** | Todas específicas com verbo + benefício + "meu" |
| **Escaneabilidade** | Hierarquia clara, bullet points, subheadings a cada 200 palavras |

### Critério de Aprovação
- Teste dos 5 segundos: aprovado por 3 personas diferentes
- Nenhum CTA genérico (Enviar, Clique aqui, etc.)
- Mínimo 1 número/dado real por seção principal
- Zero jargão corporativo sem decodificação por VOC
